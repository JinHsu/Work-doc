中断有两种，一种是**软中断**，比如代码调用 INT 指令触发。

一种是**硬件中断**，硬件通过中断控制器触发的。

但中断方式对于频繁读写磁盘数据的操作就不太友好了，会频繁打断CPU。

# Zero Copy

Zero Copy 全程不会通过 CPU 来搬运数据，所有的数据都是通过 DMA 来进行传输的，中间只需要经过2次上下文切换跟2次DMA数据拷贝，相比最原始读写方式至少速度翻倍。其实在Kafka中已经讲过Zero Copy了。
